## Tarea 2 — Corrección LCD: de I2C a paralela HD44780

### Contexto
La pantalla real es una **LCD 1602A paralela (HD44780)**, no I2C. Se eliminó toda la integración I2C y se reemplazó con la correcta.

### Archivo modificado
`firmware/econodo_esp32/econodo_esp32.ino`

### Qué se eliminó
| Elemento | Motivo |
|---|---|
| `#include <Wire.h>` | Solo necesario para I2C |
| `#include <LiquidCrystal_I2C.h>` | Librería incorrecta |
| `LiquidCrystal_I2C lcd(...)` | Objeto incorrecto |
| `#define LCD_I2C_ADDR 0x27` | No aplica a pantallas paralelas |
| `#define PIN_LCD_SDA 21` / `PIN_LCD_SCL 22` | Pines I2C, no usados |
| `Wire.begin(...)`, `lcd.init()`, `lcd.backlight()` | Métodos exclusivos de I2C |
| Todos los bloques `#if USAR_LCD` / `#endif` | Reemplazados por funciones limpias |
| `#define USAR_LCD` | Reemplazado por `const bool USAR_LCD` |

### Qué se agregó

**Librería (sin instalación adicional):**
```cpp
#include <LiquidCrystal.h>
```

**Constantes:**
```cpp
const bool    USAR_LCD = true;
const uint8_t LCD_COLS = 16;
const uint8_t LCD_ROWS = 2;
const uint8_t LCD_RS   = 13;
const uint8_t LCD_E    = 14;
const uint8_t LCD_D4   = 25;
const uint8_t LCD_D5   = 26;
const uint8_t LCD_D6   = 27;
const uint8_t LCD_D7   = 32;
```

**Objeto:**
```cpp
LiquidCrystal lcd(LCD_RS, LCD_E, LCD_D4, LCD_D5, LCD_D6, LCD_D7);
```

**Variables globales nuevas:**
```cpp
bool bmeListo        = false;
bool sdsListo        = false;
int  ultimoCodigoHTTP = 0;
int           paginaLCD       = 0;
unsigned long ultimoCambioLCD = 0;
const unsigned long INTERVALO_LCD_MS = 3000;
```

**Funciones nuevas:**
| Función | Qué hace |
|---|---|
| `inicializarLCD()` | `lcd.begin(16,2)` + `lcd.clear()` + log en Serial |
| `mostrarLCDInicio(l1, l2)` | Muestra dos líneas fijas (arranque) |
| `imprimirLCDLinea(fila, texto)` | Imprime y rellena con espacios para borrar residuos |
| `limpiarLineaLCD(fila)` | Llena fila con espacios |
| `actualizarLCD()` | Rotación no bloqueante de páginas con `millis()` |

### Conexión física de la LCD 1602A

| Pin LCD | Conectar a | GPIO ESP32 |
|---------|-----------|-----------|
| GND/VSS | GND | — |
| VDD | 5V | — |
| VO | Pin central del potenciómetro B100K | — |
| RS | GPIO 13 | 13 |
| RW | GND directo | — |
| E | GPIO 14 | 14 |
| D0–D3 | Sin conectar | — |
| D4 | GPIO 25 | 25 |
| D5 | GPIO 26 | 26 |
| D6 | GPIO 27 | 27 |
| D7 | GPIO 32 | 32 |
| A/LED+ | 5V (con resistencia 33–100 Ω) | — |
| K/LED- | GND | — |

**Potenciómetro B100K:** extremos a 5V y GND, pin central a LCD VO.

```

```

### Qué NO se modificó
- Pines BME680 (GPIO 5, 18, 19, 23) y SDS011 (GPIO 16, 17)
- Lógica de envío a Supabase
- `secrets.h`
- Cálculos de calidad de aire y estado

### Diagnóstico rápido

| Síntoma | Solución |
|---|---|
| Sin luz de fondo | Verificar 5V en VDD y GND en K |
| Luz encendida, sin texto | Girar potenciómetro de contraste (VO) |
| Cuadros negros | Contraste muy alto — girar potenciómetro al otro extremo |
| Caracteres raros | Verificar D4→25, D5→26, D6→27, D7→32 |

```html
        <h1 class="hero-titulo">EcoNodo</h1>

            ESP32 + Sensores → Supabase → Dashboard Web
        </div>
    </div>
    <div class="hero-cards">
        <!-- 3 cards: Mide, Conecta, Visualiza -->
    </div>
</section>
```

El hero se inserta **entre** el `<header>` y el `<div class="contenedor">`, fuera del layout nav+dashboard, para ser full-width.

### Diseño visual

- **Fondo:** gradiente radial bosque profundo `#0F2318 → #1A3D28`
- **Overlay:** grilla de datos con `repeating-linear-gradient` en CSS puro (opacidad 3.5%)
- **Fuente de display:** `Syne` (Google Fonts, solo para el título principal)
- **Animaciones:** `heroFadeUp` staggerada — cada elemento entra con 0.1s de diferencia
- **Nodo Supabase:** animación `pulsoNodo` (glow verde pulsante cada 2.8s)
- **Feature cards:** glassmorphism oscuro con franja de color izquierda

### Comportamiento del botón "Empezar"

- `href="#dashboard"` apunta al `<main id="dashboard">` del dashboard
- `scroll-behavior: smooth` en CSS hace el desplazamiento suave
- Sin JavaScript adicional

### Responsive

| Breakpoint | Comportamiento |
|---|---|
| Mobile (< 480px) | Columna única: contenido + cards apiladas |
| Tablet (≥ 768px) | Columna única: contenido + cards en fila |
| Desktop (≥ 1080px) | Dos columnas: contenido izq., cards der. |
